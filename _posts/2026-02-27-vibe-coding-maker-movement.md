每当新技术出现，我们总倾向于把它当作前所未有的事物——与过去一刀两断。我发现自己对 vibe coding（氛围编程/AI 辅助编程）也 constantly 有这种倾向，周围的人也一样。但理解一个新现象最有用的视角，几乎从不是这个现象本身。你需要找一些相邻的东西：足够接近以共享结构特征，又足够疏离以看得清楚。正是带着这种寻找的心态，我开始阅读更多关于 2005-2015 年创客运动（Maker Movement）的内容。

创客运动是 vibe coding 的精神前身。相似之处难以忽视。Vibe coding 有 **slop**（烂货）。创客运动有 **crapjects**——这是社区创造的一个词，指那些除了证明你能把塑料挤成某种形状外毫无用途的 3D 打印物件。那个时代的 Claude Code 是一台 200 美元的 Monoprice 打印机和一块面包板。

创客圈子产生了可能是第一批互联网原生的网络知识分子。Chris Anderson（写了那篇广泛传播的《长尾》文章）辞去《连线》主编职位，创办了一家叫 3D Robotics 的机器人公司。Cory Doctorow 写了《Makers》，一部科幻小说，描写在一个万物崩塌的世界里，角色们如何通过黑客硬件和商业模式求存。这些人通过公开参与创客文化并书写其意义而获得影响力。

AI 时代的许多智力能量围绕着 AGI：它何时到来、会对工作产生什么影响、是否会被对齐。创客运动有自己的引力中心，那就是一个理念：亲手制造实物可以带来内在的转变。你会变得更有创造力、更有创业精神、更自力更生。你制作的物品本身不如制作行为对你的改变来得重要。

2018 年，媒体学者 Fred Turner 发表了一篇论文，将这种意识形态置于显微镜下。他的论点是：创客运动为数字时代重新发明了西部边疆的神学。

17 世纪清教主义的具体内容当然消失了。创客博览会上没人谈论预定论。但 Turner 追踪了文学形式和千禧年结构——那种相信伟大变革即将到来、个人修行将决定谁能闯过难关的信念。在创客叙事中，美国的经济景观是贫瘠的。工作消失了。机构辜负了你。在这片荒原中，孤独的个人在内心寻找创业精神的迹象、创造力的火花——证明自己是那些将建造通往救赎之路的选民之一。

Turner 的观察远超 3D 打印机。你可以在过去五十年的几乎每一个业余技术场景中追溯同样的模式。1970 年的家酿计算机俱乐部。1980 年的朋客杂志。1990 年的早期互联网。每一个都发展出了一个实践社区——Brian Eno 会称之为「scenius」（场景天才）——人们在其中玩弄主流认为是玩具的工具。每一个都产生了自己的救赎叙事：掌握这个工具，转变自己，成为建造未来的人。

每一个都有一个有用的「松弛」空间。这些工具故意是非生产性的。没人指望你的 Arduino 项目发货给客户。没人指望你的家酿计算机与 IBM 竞争。关键在于你有许可去折腾，而发现是逐渐发生的，通过游戏，经过多年。这就是硅谷那句老话的来源：「聪明人周末做的事，十年后其他人会在工作日做。」

**Vibe coding 以一种重要的方式打破了这个模式。**

之前的每一波业余技术浪潮都经历了一个 scenius 阶段——一个奇怪的群体在任何人期待经济产出之前玩弄工具的时期。

**Vibe coding 完全跳过了这个阶段。** 它被直接部署给普通大众，几乎立即进入企业公司的代码库和完善的产品中。没有受保护的游戏场时期。没有时间积累 scenius 社区产生的那些奇怪的、无用的、好玩的知识。相反，立即面临一次性打造爆款产品或首次尝试就解决复杂用例的压力。

这很重要，因为 scenius 阶段正是内在转变实际发生的地方。当你花两年时间制作无用的 Arduino 项目时，你发展出了关于电子、材料和设计的直觉——这是教程无法给你的。当 vibe coding 直接进入生产时，你失去了那个发展空间。工具足够强大，可以在使用者发展出真正的判断力之前产生真正的输出。当我与每天在 Claude Code 上待 12-14 小时的人交谈时，我感觉自己在与被某种东西附体的人交谈，试图把握一个不同的现实。在 scenius 的情况下，将你拴在现实的反馈回路由其他人提供。有人看着你的项目告诉你它毫无意义，或者精彩绝伦，或者两者皆是。而在 vibe coding 的情况下，反馈回路由机器提供，你不断试图辨别自己是疯了还是真的产生了有价值的东西。

它产生的东西类似于轻躁狂（hypomania）：一种你的生产能力真正提高的状态。你不是在想象自己完成了更多——你确实完成了更多——但你的评估能力不习惯这种创作模式。你失去了区分「这很好」和「我制作这个感觉很好」的能力。一切都感觉像是突破。输出是真实的，但你与它的关系是扭曲的。

Vibe coding 的速度和轻松创造了一种评估麻醉。你无法判断自己是否构建了有用的东西，还是只是构建了某种存在的东西。在某种程度上，这是 60 年代嬉皮士第一次尝试 LSD 的清醒版本：有时你可能有突破，有时你可能崩溃，但无论哪种，这都是 Fred Turner 所说的通过制造获得救赎的反面。

旧的「通过制造转变」隐喻不适合 vibe coding 还有第二个原因，这与创客运动实际上如何结束有关。

核心承诺——分布式数字制造将把制造业带回美国、每个城市都会有微型工厂、3D 打印将去中心化生产——根本没有实现。实际发生的事情遵循了 Joel Spolsky 多年前在他的《商品化你的互补品》一文中描述的模式：廉价的 3D 打印机和 Arduino 使**原型制作**几乎免费，这确实有用。但关于如何实际大规模制造事物的深层、复利知识继续在像深圳这样的工业基地积累。原型制作被民主化了。廉价工具商品化了技术栈的一层，使其下面的一层相比之下更有价值。

你现在可以看到结构相似的事情在 vibe coding 上发生。人们正在快速原型化威胁到整个 SaaS 商业模式的工具。但所有这些快速迭代和原型制作产生的价值向上流动。它在模型层、训练数据、基础设施中积累。Vibe 编码者自己面临着变得可互换的风险，每个人都在推出令人印象深刻的演示，却没有积累自己持久的价值。模式相似：廉价工具民主化一层，下面的一层捕获剩余。

在这两种力量同时作用的情况下——没有 scenius 阶段来发展，价值向上游积累而不是与制造者在一起——旧的通过制造转变的隐喻不完全适用了。我们需要一个新的隐喻。

**我提供的隐喻是：消费。**

具体来说：对过剩智能的消费。AI 代表着大量可用的认知能量，vibe coding 是在浪费这种能量之前消耗它的一种方式。把它想象成一种正在产生的资源，无论你是否使用——而 vibe coding 是将这种剩余引导到游戏、探索、快速创造的行为中，这些可能产生也可能不产生持久的产物。

这种框架已经开始出现在各个地方。Rachel Thomas 将 vibe coding 某物的体验比作赌博时的黑暗心流状态。意思是说你正在对创造的表面体验上瘾，这可能开始时是心流，但最终变成你上瘾的东西，而不是帮助你成长的东西。

消费几乎总是被视为负面行为，尤其是如果你是创业者或建设者。消费是被动者做的事。建设者生产。

我认为这个框架是错误的，或者至少是不完整的。有几种富有成效的方式来思考消费实际上产生什么。

**当生产变得闪电般快速且边际成本低廉时（当你可以在一个下午开发出一个应用），稀缺资源转向知道什么应该存在。** 那个烧掉几十个原型、构建东西并立即丢弃的 vibe 编码者，发展出一种模型本身不具备的模式识别能力。这是关于什么值得构建、什么感觉对、用户真正想要什么的判断。这是一种感性，而感性出了名难以商品化，因为它是难以解读的。你通过制作很多东西并关注哪些感觉有活力、哪些感觉死气沉沉来发展它。

这里的价值捕获看起来像创意指导、策展、品味塑造、顾问角色。你在出售通过制作东西并扔掉它们而发展的辨别力。点子人回来了。在这条路的极端，你成为 William Gibson 的《模式识别》的主角：一个审美直觉如此精细的人，公司雇佣他们仅仅是为了对已经可以生产的东西说是或否。

**可见的消耗产生奇观，奇观产生注意力。** 当你公开 vibe 编码某物时——快速构建、立即发布、在观众面前迭代——你制作的产品不如制作的表演重要。毫无疑问，今天的 vibe coding 很大一部分是纯粹的信号表演。

最近的「一个周末建成」帖子浪潮就是这个原理。产品往往很一般。有时简直是可抛弃的。但制作它、把握发布时机、在正确时刻将其投入网络的行为是一种剩余的表演，人们观看表演。价值捕获是观众、声誉，以及这些以未来合作、工作邀约、投资者兴趣、咨询机会形式创造的期权。

这在结构上与内容创作者已经运营的方式相同。YouTuber 的单个视频是一种消耗。跨数百个视频积累的观众是资产。Vibe coding 只是给内容创作者的工具包增加了另一种媒介：你不再把精力花在文章或视频上，而是花在应用和工具上，以同样的方式捕获注意力。

**如果你把 vibe 编码的产出视为礼物——开源工具、免费工具、共享模板、公开仓库——你正在创造条件来占据网络中有趣或强大的位置。** 想想构建早期互联网最有用的免费工具和资源的人：他们成为了其他人围绕定位的节点。

礼物经济一直是开源的潜在价值捕获策略，但消费框架解释了为什么它在心理上对 vibe 编码者有效，而通常的「构建开源项目以获得雇佣」建议不太能引起共鸣。当你把它框架为战略性职业建设时，它感觉交易性且有点绝望。当你把它框架为消耗剩余时，它感觉自然。你有通过这些工具可用的额外认知能量。你花掉它。你送掉你制作的东西。礼物经济做它一直做的事：它创造社会联系、声誉和互惠义务。

**每次你 vibe 编码某物时，你都在产生信号。** 关于用户想要什么的信号。关于哪些模式有效的信号。关于模型在哪里失败、它错过了什么边缘情况、它误解了什么指令的信号。这些信号目前免费向上游流向模型提供商。你的提示、你的迭代、你的修正——所有这些都成为下一代模型的训练数据。在非常字面的意义上，每次你构建某物时，你都在为基础设施层执行无偿劳动。

但信息废气可以在向上游飘散之前被捕获。如果你能构建你产生的信号——作为专有数据集、作为文档化的反馈回路、作为在特定领域什么有效什么无效的系统记录——你最终会持有基础设施层真正需要且无法轻易复制的东西。每次 vibe coding 会话都产生这种废气作为副产品。问题是你让它消散还是收集它。收集它的人最终构建了你可能称之为数据堡垒的东西：一个随着每个原型变得更强的位置，即使是那些被扔掉的原型，因为它们为什么失败的知识是有价值的部分。

这就是早期创客在 scenius 中完成的事情的精神。尽管他们的输出可能微不足道，但他们沉浸在生产过程中，通过那样发展了对他们媒介的触觉理解。有了 vibe coding，这些数据是免费生成的。你会使用它吗？

***

消费不必是被动的。剩余可以被好好花费。关键区别在于你是否带着某种对燃烧产生什么的意识来燃烧能量——品味、注意力、社会资本、结构化信号——还是你只是启动十几个项目然后纳闷为什么没有一个能粘住。

就我个人而言，我发现消费隐喻是应对不断使用 AI 做各种事情带来的倦怠的好方法。很多人以工艺的心态来制作东西，并自然地将这个框架延伸到 vibe coding。那个框架感觉高尚且深深熟悉，但它也是倦怠的配方，因为工艺假设你在向内探索并从中提取某物。工艺的整个情感架构是转化性的：你挣扎，发展精通，你生产的物品是内在改变的证据。当工具完成大部分生产时，那个框架开始崩塌。你被留在一个从未要求你发展某物的过程中向内探索，你预期投入的努力与实际需要的努力之间的差距开始感觉像是个人失败而不是技术的特征。

消费框架完全回避了这一点。你不是向内探索。你从一个有额外能量可用且需要去某处的位置开始。问题从「这对作为制造者的我意味着什么」转变为「我可以把这个花在最有趣的事情上是什么」。那是一种根本不同的情感姿态，在实践中它是一个更加可持续的姿态。

## 核心要点

1. **历史会重演**：Vibe coding 与创客运动有惊人的结构相似性，包括大量低质量产出（slop vs crapjects）和相似的神学框架。

2. **缺失的 Scenius 阶段**：与之前的技术浪潮不同，vibe coding 跳过了社区孵化期，直接进入生产环境，导致使用者缺乏发展判断力的空间。

3. **新的隐喻：消费而非创造**：与其把 vibe coding 理解为「通过制造获得转变」，不如把它理解为「对过剩智能的消费」。

4. **价值捕获的四种路径**：
   - 培养品味和判断力（成为创意指导/顾问）
   - 产生奇观获取注意力（内容创作者模式）
   - 礼物经济（开源社区模式）
   - 捕获信号建立数据堡垒

5. **对抗倦怠**：用「消费」而非「工艺」的框架理解 vibe coding，可以更可持续地使用这些工具。
