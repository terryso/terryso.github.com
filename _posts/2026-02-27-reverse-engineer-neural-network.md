很多「夺旗赛」风格的机器学习谜题都会给你一个黑盒神经网络，你的任务是猜出它做什么。但当我们去年年初考虑创建自己的 ML 谜题时，我们想做点不一样的。

我们觉得给用户一个**完整的神经网络规格**——包括所有权重——会很酷。这样他们就必须使用**机制可解释性（mechanistic interpretability）**的工具来逆向工程这个网络。这正是我们在研究中有时会遇到的情境：试图解释复杂模型的特性。

去年二月我们发布了这个谜题。当时我们甚至不确定它是否有解。我们设计的神经网络对几乎所有输入都输出 0。合理的求解者可能会假设目标是找到一个能产生 1 或其他非零值的输入。但我们设计这个网络的方式（你很快就会看到），让你无法用传统方法暴力破解——比如通过反向传播将非零输出一直传回输入层。你必须**真正理解这个网络在做什么**。

我们对这个谜题的反响感到惊讶。主要是运气好，我们的难度校准恰到好处：既没有难到没人能解决，也没有简单到被答案淹没。事实上，如果你能解开这个谜题，你很有可能适合 Jane Street。

下面我们会重述这个问题，但要注意：**这篇文章包含大量剧透**。如果你想自己尝试解谜，请移开视线。本文将讲述一位真正解题者的过程，包括所有曲折，直到最终破解。

## 问题

> 今天我去远足，在一堆新石器时代的墓葬堆下面发现了一堆张量！我把它们送到当地的神经网络管道工那里，他们设法拼凑出了这个。
>
> `model.pt`
>
> 总之，我还不确定它做什么，但对这个古老的文明来说一定很重要。也许可以从最后两层开始看。
>
> 模型输入：`vegetable dog`
>
> 模型输出：`0`
>
> 如果你弄明白了，请告诉我们。

那个 `model.pt` 文件基本上就是一个 pickled PyTorch 模型。

## 解题过程

### 入门

Alex 是一名大四学生，室友告诉他一个在 Twitter 上流传的谜题时，他正在宿舍里。室友自己试过，两个晚上后放弃了。Alex 在学校的最后一个冬天，正想找点事做，决定试试。

他先下载模型，四处查看，特别关注最后一层：

```python
import torch
import plotly.express as px

model = torch.load('./model.pt')
linears = [x for x in model if isinstance(x, torch.nn.Linear)]
px.imshow(linears[-1].weight.detach())
```

![最后一层权重](https://blog.janestreet.com/image6.png)

**立刻就很明显**：这不是一个普通的神经网络。它显然没有经过训练——所有权重都是**整数值**。相反，它是手工设计的，可能用于执行某些非常特定的计算。

最后一层是一个 48x1 矩阵，但明显分为三个部分。如果你看前一层的激活值，它们总是三份相同的东西的重复。倒数第二层似乎是三份相同权重的重复，而它的偏置包含相同的 16 字节，但每次增加 1，就像编码一个向量 v，然后是 v+1，和 v+2。

```python
px.imshow(linears[-1].weight.detach())
```

![倒数第二层权重](https://blog.janestreet.com/image3.png)

```python
px.imshow(linears[-2].bias.detach().unsqueeze(0))
```

![偏置](https://blog.janestreet.com/image1.png)

思考了一下——以及最后一层输出单个比特的事实——Alex 意识到这个倒数第二层的 ReLU 必须是在计算两个 16 字节整数是否相等（每个神经元一个字节）。它的工作方式是制作输入向量 v（一个 16 字节数字）的三份副本，然后尝试与一个参考数字 x（由倒数第二层的偏置决定）进行比对。所以三份副本实际上代表 v-x-1、v-x 和 v-x+1。最后一层分别对这些情况应用权重 1、-2 和 1。

最后一层的偏置是 -15，所以只有当 v=x 对所有 16 字节都成立时，最后的神经元才会被激活。

**所以现在问题变成了：如何让倒数第二层的激活值等于 x？**

### 逆向工程网络核心的程序

Alex 想，如果网络最后在检查某个数字，那网络的其余部分一定是某种大方程。确实，网络中似乎有很多结构，你可以从绘制 2500 个线性层（约占整个网络的一半）的大小看出：

```python
px.line([l.out_features for l in linears])
```

![层大小](https://blog.janestreet.com/image2.png)

于是 Alex 开始查看各种子网络，追踪它们的依赖关系。这涉及盯着很多图结构：

![图结构1](https://blog.janestreet.com/image4.png)
![图结构2](https://blog.janestreet.com/image5.png)

但经过几个小时的搜索，他一无所获。目前看来，手动追踪的复杂性太高了。于是他有了一个新想法：**如果我把这东西当作一个线性规划来求解呢？**

当然，对于这么多 ReLU 层这是不可能的——ReLU 不是线性的——但可以通过添加一个额外的**整数值**来建模，对应于「这个激活值是负的」这个陈述。因此可以把它当作整数线性规划，使用支持整数规划的约束求解器。Alex 就这么做了：他认真地写了一些代码，将神经网络的各层转换成一个巨大的线性规划，然后让它运行。

然后一直运行。

看起来毫无进展——于是 Alex 现在试图减少程序中的变量数量。也许可以做一些简化？

Alex 发现，如果你看一堆层，它们大多看起来像单位矩阵。实际上，在大约 1500 层中，80% 的节点只是在执行恒等操作。

Alex 将网络中的每个神经元视为 DAG 中的一个节点，每个节点以某些权重进入下一层的节点；但如果你有一个入度为 1 且权重恰好为 1 的节点，你可以合并这两个节点。（你知道这样做是安全的，因为网络到处都是整数值：所有输入都是整数，所有权重也是。）

还有一些稍微花哨的简化。例如，如果你有一个节点的每个入边都有正权重，那么你做 ReLU 就不重要了，因为它永远不会触及负值钳位——所以你可以将它的入边直接转发给孩子节点，直接传递到下一层。另外，如果一层中的两个神经元有完全相同的输入向量，你可以合并它们，并将它们的后代重定向到新的合并神经元。你可以多次重复这个过程。

Alex 现在已经在这个分析上投入了数小时。他发现了似乎在许多层中重复出现的电路。他会打印出不同的节点等价类，查看每个节点作为输入的权重序列，发现只有几**种**节点。例如，有一类节点实际上会将值从两层前转发。合并这些以及其他类似的简化，将线性规划的大小从大约 200 万个节点减少到 75,000 个。

但经过这一切，Alex 再次运行求解器，它仍然没有终止就一直在运转。

### 最后的简化

新想法：**如果你通过网络传播边界呢？** 只需逐层推理，你可以算出任何给定节点可以达到的最大值；你只需查看其输入的边界。结果发现，在相当保守的假设下，许多节点最终会有非常紧的边界，例如 0-1。也许这足以使程序变得可处理？

此时 Alex 从线性规划切换到 SAT 求解器，因为值的总数已经变得小得多。在 SAT 版本中，每个节点等于其范围内的每个值都有一个布尔变量。总共，经过所有简化后，这产生了 200,000 个变量。运行一天后，SAT 求解器将程序减少到 20,000 个变量。从那里开始，它似乎不再减少了。

实际上，Alex 发现这个神经网络内部有一个**核心程序**，**不可约地复杂**——让他非常失望的是，它仍然太大，无法暴力破解。所以经过许多天，他不得不退一步，实际上没有任何进展。

### 与解擦肩而过

他进行元思考：这必须是一个可解的谜题，对吧？怎么会有人构建一个**有趣**可解的谜题？如果你生成随机权重，SAT 求解器可能能够通过暴力破解。这个网络是由人类创建的。在其核心，似乎有一个你无法仅通过搜索或优化来恢复的函数。这是一个**不可逆**函数。有哪些常见的不可逆函数？

Alex 问 ChatGPT 一些常见的哈希函数，并将它们与层宽度的一些基本图进行比较，看起来是周期性的。实际上有 32 个长度为 48 的周期，每次精确重复。也许网络在做 32 块相同的计算？再问 ChatGPT：有没有使用 32 块计算的常见哈希函数？**中奖了**。结果几乎所有这些都这样做。

为了确定是哪一个，他手动探索：他向网络输入一些字符串，用单独的程序计算各种哈希变体，然后查看倒数第二层。结果 **md5** 对齐了，而其他常见的哈希函数没有。

这很好，因为他已经知道哈希应该是什么——通过查看倒数第二层的偏置。所以问题简化为**找到一个能产生那个特定 md5 哈希的输入字符串**。但如何解决这一点并不明显——特别是因为他没有真正的证明这个网络**总是**产生 md5 哈希。也许解决方案是更深入地挖掘，并破解网络使其可逆？

### 矩阵中的故障

Alex 注意到网络中有些奇怪的东西。它似乎有一个 bug：如果你的输入长度大于 32，它就不再产生正确的 md5 哈希。也许那个 bug 中有一个密钥可以逆转内置在网络中的哈希值？

他花了接下来的两天逆向工程这个 bug。首先，他让 Gemini 写了一个 md5 哈希函数的实现。然后他将网络中的每个神经元与 md5 算法中的相应变量匹配起来。他写了一些代码，存储给定中间变量的值序列，然后在网络的 32 个块中搜索该值；这将挑选出哪些范围的神经元对应于每个变量的位。结果发现，一些位范围正好对应于变量，而其他是中间计算值。

然后，对于长度 >32 的输入，他可以费力地追踪各个块，找到网络偏离正确算法的确切位置。

**关键在于前 7 层**——有一个电路会计算输入的长度，并尝试以小端顺序将其存储在 4 个字节中。但当长度为 256 位或更大时，你会得到一个包含**值** 256 的长度变量，而不是正确的编码。也就是说，如果长度 >384 位，长度字节应该是 `128 1 0 0`，但网络编码的却是 `384 0 0 0`。

这花了大约两天时间弄清楚，但——又一次失望——没有让 Alex 更接近解决方案。他写信给谜题提供的电子邮件地址，告诉他们他发现的东西。他听到的回复让他惊讶。那个 bug **不是故意的**。考虑到这一点，为什么不最后再试一次呢？

### 暴力破解的回归

**结果发现**，一旦你知道了编码在倒数第二层偏置中的哈希，你**就完成了**。弄清楚这一点是谜题的核心。谜题创建者故意使哈希易于暴力破解，在谜题描述和 Python 代码中留下了各种小暗示，表明**解决方案由两个英文单词组成，小写，用空格连接**。

Alex 实际上早些时候尝试过暴力破解哈希，但他下载了一个前 10,000 个最常用单词的列表来做这件事，结果发现不够大。一旦他有了足够大的单词列表，他得到了答案。

> **译者注**：这个答案就是题目中的示例输入 `vegetable dog`。这个神经网络实际上是一个 MD5 哈希检查器——它会计算输入的 MD5 哈希，然后检查是否等于某个特定值。如果相等，输出 1；否则输出 0。谜题的核心挑战是识别出这个网络在实现 MD5 算法，然后从偏置中读取目标哈希值，最后通过暴力破解找到原始输入。

## 另一个谜题

使这个谜题具有挑战性的因素之一是设计一个具有正确复杂性的网络。使用逻辑门意味着网络不可微分；但如果你让这些门编码的程序太复杂，就几乎没有逆向工程的希望。MD5 感觉是一个不错的折衷，尽管它绝非易事。因为 MD5 使用模加法，创建谜题需要在神经网络的 20 多层中实现一个**并行进位加法器（parallel carry adder）**。不容易！我们印象深刻的是一些求解者设法弄明白了——而 Alex 发现 >32 的 bug 是出乎意料且相当非凡的。

创建和发布谜题，以及与解决它的人们互动的体验足够好，我们又做了一次。这里你会找到[最新的](https://www.janestreet.com/neural-network-puzzle-2/)。在这个新谜题中，一个层的顺序被打乱的神经网络需要被放回正确的顺序……你能帮忙吗？

## 总结

这篇文章展示了一个非常巧妙的机器学习谜题设计：

1. **反传统的谜题设计**：不是给你黑盒让你猜，而是给你完整的权重让你逆向工程
2. **手工设计的神经网络**：使用整数权重实现特定的算法（MD5 哈希）
3. **多种解题策略**：从线性规划、SAT 求解器到哈希函数识别
4. **意外的深度**：求解者发现的 bug 甚至连出题者都不知道

**关键启示**：机制可解释性（mechanistic interpretability）是一个非常有价值的领域。理解神经网络内部在做什么，而不仅仅是训练它，是 AI 安全和可解释性研究的重要组成部分。
