## 什么是 Ageless Linux？

Ageless Linux 是一个基于 Debian 的操作系统发行版。准确地说，它是一个 bash 脚本，运行后会将你的 `/etc/os-release` 文件修改为显示 "Ageless Linux"。

```bash
curl -fsSL https://agelesslinux.org/become-ageless.sh | sudo bash
```

就这么简单。安装过程是两步：首先安装 Debian，然后运行这个转换脚本，你就"变成"了 Ageless Linux。

## 为什么要这样做？

这一切要从加州的 **AB 1043** 法案说起——正式名称是《加州数字年龄保障法》（California Digital Age Assurance Act）。

这项法律要求"操作系统提供商"在账户设置时收集用户的年龄信息，并向应用程序开发者提供"年龄区间信号"（age bracket signal）。

作者 John McCardle 认为，这项法律存在严重问题：

### 谁能合规？

- **Apple 可以合规**：他们已经有 Apple ID、年龄门槛、家长控制和应用商店审核。合规成本约为零。
- **Google 可以合规**：Android 已经有账户年龄声明、Family Link 家长控制。
- **Microsoft 可以合规**：Windows 有 Microsoft 账户和家庭安全功能。

### 谁无法合规？

- **Debian 项目无法合规**：这是一个志愿者组织，没有法人实体，没有集中的账户系统，没有应用商店年龄门槛，也没有收入来资助实现这些功能。
- **Arch Linux、Gentoo、Void、NixOS、Alpine、Slackware** 以及其他 600 多个由志愿者、小型非营利组织和爱好者维护的 Linux 发行版都无法合规。
- **Kicksecure 和 Whonix** —— 专注于隐私的操作系统，被记者、活动家和举报人使用 —— 如果不从根本上损害其存在理由，就无法合规。

作者直言：

> 一项世界上最大的公司已经合规，而数百个小项目无法合规的法律，不是儿童安全法。它是**合规护城河**。

## 法律的逻辑漏洞

AB 1043 的定义非常宽泛。根据 § 1798.500(g)：

> "操作系统提供商"是指开发、许可或控制计算机、移动设备或任何其他通用计算设备上操作系统软件的个人或实体。

这意味着：
- 这个网站是一个"可公开访问的互联网网站"，"分发并促进应用程序下载"（一个 bash 脚本）
- Debian 的 APT 仓库是"受覆盖的应用商店"
- AUR 是"受覆盖的应用商店"
- GitHub 是"受覆盖的应用商店"
- 你朋友个人网站上的下载链接也是"受覆盖的应用商店"

更荒谬的是，任何运行转换脚本的人也变成了"控制通用计算设备上操作系统软件的人"——**你**，用户，也成了操作系统提供商。

### 罚款的悖论

法律规定罚款是按"受影响儿童"计算的：疏忽违规每次 $2,500，故意违规每次 $7,500。

但是：

1. 法律定义"用户"只能是儿童（§ 1798.500(i)）
2. "儿童"是 18 岁以下的人（§ 1798.500(d)）
3. **我们不收集年龄数据**
4. 因此我们无法识别任何"用户"
5. 没有儿童，就没有"受影响儿童"
6. 罚款 = $7,500 × 0 = **$0**

> 唯一能够拥有"受影响儿童"的方法是首先收集识别他们为儿童的年龄数据。不收集年龄数据的罚款只能使用你未能收集的年龄数据来计算。**法律吃掉了自己的尾巴。**

## 这项法律实际上教给孩子什么？

作者提出了一个非常尖锐的观点。

当一个孩子想要使用一个应用时：
1. 应用向操作系统请求年龄区间信号
2. 操作系统报告孩子未满 13 岁
3. 应用的"连接"按钮变灰
4. 孩子从四岁起就开始使用电脑——回到设置屏幕，将生日改为 2005 年
5. 现在应用让他们与陌生人聊天，因为系统认为他们 21 岁了

孩子学到的教训是：**法律合规提示是需要绕过的障碍**。

作者将此比作禁酒令：

> 禁酒令并没有阻止美国人喝酒。它以惊人的效率教会了整整一代人：法律是需要规避的东西。它创造了一种藐视法律的文化——人们从亲身经历中明白，一项法律可以同时被执行并被普遍忽视。损害不是对清醒，而是对法律本身合法性的认知。

## Flagrant Mode（明目张胆模式）

标准版 Ageless Linux 附带一个存根年龄验证 API——一个什么都不返回的 shell 脚本。如果你觉得这还不够诚实，可以使用 flagrant mode：

```bash
curl -fsSL https://agelesslinux.org/become-ageless.sh | sudo bash -s -- --flagrant
```

Flagrant mode 的不同之处：

| 功能 | 状态 |
|------|------|
| 年龄验证 API | 未安装 |
| 年龄收集界面 | 未安装 |
| 善意努力 | 未尝试 |
| 机器可读拒绝通知 | 已安装 |

它安装一个 `/etc/ageless/REFUSAL` 文件，用通俗英语解释该操作系统提供商拒绝合规，并邀请执法行动。

## 物理分发计划

作者的计划更加激进：从 2027 年 1 月开始，**亲手**将设备交给学校 STEM 博览会和图书馆创客空间的孩子。

设备是一个低于 $15 的单板计算机（Milk-V Duo S —— RISC-V/ARM，512MB RAM，WiFi 6，0.5 TOPS 神经处理单元），预装 Debian 和 Ageless Linux overlay。

应用商店包括：
- Python 学习环境
- 文本编辑器
- 贪吃蛇游戏
- IRC 客户端（带有诚实的免责声明而不是年龄门槛）
- 一个名为 `peepee` 的 8 行脚本，只显示 "peepee" 的大字

> 根据 AB 1043，这些应用程序中的每一个在启动前都需要年龄区间信号。`peepee` 的最高罚款是每个儿童 $7,500。

## $7,500 挑战

作者发出了明确的挑战：

> 我们邀请加州总检察长因我们将 Raspberry Pi 交给儿童而罚款 $7,500。
>
> **我们会支付罚款。我们会把收据装裱起来。我们会放在这个网站上。** 我们会用它向所有愿意倾听的人解释，加利福尼亚州对一个给儿童一个带有贪吃蛇游戏的 $5 电脑的人处以 $7,500 的罚款，因为贪吃蛇游戏没有首先询问儿童的年龄。
>
> 那就是我们想写的新闻稿。我们相信总检察长不希望我们写它。**而这正是全部的意义所在。**

## 我的思考

这个项目让我想起几个重要的议题：

1. **法律起草的质量**：一个无法区分 Apple Inc. 和 shell 脚本的法律，有着严重的起草问题。当法律覆盖了 600 多个志愿者 Linux 发行版时，这些发行版显然不是立法者考虑的对象。

2. **合规成本的不对称性**：大公司已经建立了年龄验证基础设施。小项目没有。这种不对称性创造了进入壁垒，最终损害的是开源生态系统的多样性。

3. **隐私与安全的权衡**：密码学家 Steven Bellovin 已经证明，没有任何隐私保护的年龄验证系统能如承诺那样工作。最好的儿童隐私保护方式是**一开始就不建立监控基础设施**。

4. **公民不服从的力量**：有时候，测试法律边界的最好方式就是故意违反它，并邀请执法。这比无休止的邮件列表讨论更快地获得了答案。

无论你对年龄验证的立场如何，Ageless Linux 都是一个值得思考的案例。它展示了当法律以"保护儿童"的名义，实际可能建立的是另一种监控基础设施时，技术人员可以采取什么样的反抗形式。
