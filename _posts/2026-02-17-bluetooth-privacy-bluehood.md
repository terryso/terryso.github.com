如果你经常读我的博客，会知道我对隐私问题特别关注。无论是通过 Tor 运行博客、用 AdGuard 在网络层面屏蔽广告，还是用 Proton Pass 把敏感信息从 dotfiles 里隔离出去——我总是会仔细思考自己暴露了什么数据、暴露给了谁。

上周末我开发了一个叫 **Bluehood** 的蓝牙扫描工具，它可以追踪附近的设备并分析它们的出现模式。这个项目大量借助了 AI 的力量，但动机完全是人类的：我想弄清楚，仅仅是开启蓝牙，我到底泄露了多少信息。

时机也很巧。几天前，KU Leuven 的研究人员披露了 **WhisperPair**（CVE-2025-36911）——一个影响数亿蓝牙音频设备的严重漏洞。攻击者可以利用这个漏洞远程劫持耳机、窃听对话，甚至通过 Google 的 Find Hub 网络追踪位置。这再次提醒我们：蓝牙并不是我们想象中那种隐形、无害的信号。

## 大家都不愿谈论的问题

我们已经习惯了"蓝牙永远开着"。手机、笔记本、智能手表、耳机、汽车，甚至医疗设备，都在不断广播自己的存在。对于隐私担忧，标准回答通常是"身正不怕影子斜"。

但问题是：即使你没什么可隐瞒的，你仍然在泄露你可能不想泄露的信息。

我在家里的办公室运行 Bluehood（被动模式，只监听，从不连接），可以检测到：

- 送货车辆什么时候到达，每次是不是同一个司机
- 邻居们基于手机和可穿戴设备的日常活动规律
- 哪些设备总是一起出现（比如某人的手机和智能手表）
- 某些人具体什么时候在家、在公司、或者在别处

这些都不需要任何特殊设备。一个带蓝牙适配器的树莓派就够了。大多数笔记本也可以。

## 你无法控制的设备

最让我担心的不是人们主动选择开启蓝牙，而是很多设备根本不给用户关闭的选项。

助听器就是一个例子。现代助听器通常使用 Bluetooth Low Energy（BLE），这样听力专家可以连接设备调整设置或运行诊断。起搏器和其他植入式医疗设备有时也会广播 BLE 信号，原因相同。用户没法简单地关闭它。

还有车辆。送货车、警车、救护车、物流车队、火车——它们通常有用于车队管理、诊断或驾驶辅助的蓝牙系统。这些系统持续广播，司机对此无能为力。

即使是消费级设备也不总是那么直接。很多智能手表必须依赖蓝牙才能正常工作。宠物 GPS 项圈需要它来与主人的手机通信。某些健身器材没有它就完全无法使用。

## 需要你广播的隐私工具

有趣的是，一些最注重隐私的项目恰恰需要开启蓝牙。

**Briar** 是一个点对点消息应用，专为在敌对环境中工作的活动家和记者设计。它不依赖中央服务器，当互联网断开时，可以通过蓝牙或 Wi-Fi mesh 网络同步消息。这对于在互联网管制或监控严重的地区维持通信非常有用。

**BitChat** 更进一步。这是一个完全通过蓝牙 mesh 网络运行的去中心化消息应用——不需要互联网，不需要服务器，不需要电话号码。每台设备既是客户端也是中继，自动发现附近的其他设备，并通过多跳转发消息来扩展网络覆盖范围。这个项目明确针对抗议活动、自然灾害、以及网络受限或审查地区的场景。

这两个都是真正优秀的项目，解决着真实的问题。但要使用它们，你需要开启蓝牙。而每个开启蓝牙的设备都在向附近任何愿意监听的人广播自己的存在。

这就产生了一个奇怪的矛盾：为保护隐私而设计的工具，往往需要一项在其他方面会损害隐私的功能。

## 人们低估了模式的力量

人们常常低估行为模式能揭示什么。一个拿着蓝牙扫描器的恶意行为者不需要知道你的名字，他们只需要观察时间。

想想看，如果有人在居民区监听几周蓝牙信号，能了解到什么：

- 这家人通常什么时候不在家？
- 是否有人每周四下午都会来访？
- 是否存在某种规律性，暗示着倒班工作？
- 孩子们什么时候放学回家？
- 哪些家庭有相同的送货司机（暗示相似的购物习惯）？

如果你家的财产遭到损坏，你甚至可以回溯日志，看看那个时间点附近有哪些设备。一个遛狗人手腕上的智能手表。某人裤子口袋里的手机。一辆有车队追踪系统的车辆。

这些听起来可能是极端情况，但它们说明了一个更广泛的观点：我们不断留下自己甚至没想过的数字痕迹。

## Bluehood 实际上做什么

Bluehood 是一个 Python 应用，可以在任何有蓝牙适配器的设备上运行。它持续扫描附近的设备，通过厂商和 BLE 服务 UUID 识别它们，并追踪它们的出现和消失时间。

主要功能：

- **被动扫描**：只监听，不尝试连接或与任何设备交互
- **设备分类**：通过 BLE 指纹识别手机、音频设备、可穿戴设备、车辆、IoT 设备等
- **模式分析**：每小时的设备活动热力图、驻留时间追踪、关联设备检测
- **过滤**：检测并隐藏随机化 MAC 地址（现代手机用于隐私保护的功能）
- **Web 仪表盘**：简单的监控和分析界面

你可以用 Docker 运行，也可以直接安装。数据存储在 SQLite 中，还可以选择通过 ntfy.sh 发送推送通知，当被监视的设备到达或离开时提醒你。

## 如何运行

最简单的尝试方式是用 Docker：

```bash
git clone https://github.com/dannymcc/bluehood.git
cd bluehood
docker compose up -d
```

仪表盘地址是 `http://localhost:8080`。

如果你更喜欢手动安装：

```bash
sudo pacman -S bluez bluez-utils python-pip  # Arch
sudo apt install bluez python3-pip           # Debian/Ubuntu
pip install -e .
sudo bluehood
```

蓝牙扫描需要提升权限。你可以以 root 运行、给 Python 授予 capabilities，或者使用附带的 systemd 服务来实现常驻监控。

## 这一切的意义

Bluehood 不是黑客工具。它是一个教育性的演示，展示了用普通硬件和一点耐心能做到什么。

我做它是因为我想亲眼看看自己在广播什么。结果令人警醒。即使没有任何恶意企图，任何有基本技术知识的人只要坐在车里运行一个脚本，就能对我的家庭了解很多。

这不是关于偏执。这是关于理解我们在设备上保持无线射频开启时所做出的权衡。对于某些用例，蓝牙是必不可少的。对于其他用例，它只是便利。意识到自己在暴露什么，是做出明智决策的第一步——判断你的设备属于哪一类。

如果你尝试了 Bluehood，它让你对自己的蓝牙习惯三思而后行，那它就完成了使命。

## 关键要点

1. **蓝牙比你想象的更暴露**：即使没有恶意意图，被动监听蓝牙信号也能揭示大量关于你日常生活的信息
2. **很多设备无法关闭蓝牙**：从助听器到物流车辆，用户对蓝牙广播往往没有控制权
3. **隐私工具的矛盾**：一些隐私保护工具（如 Briar、BitChat）恰恰需要蓝牙才能工作
4. **模式识别的力量**：长期监听可以发现作息规律、社交关系、甚至安全隐患
5. **意识是第一步**：Bluehood 的目的不是攻击，而是让更多人意识到自己正在广播什么
