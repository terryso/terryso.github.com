今天，OpenAI 发布了 **GPT-5.4**，该模型已上线 ChatGPT（作为 GPT-5.4 Thinking）、API 和 Codex。这是 OpenAI 迄今为止最强大、最高效的专业级前沿模型。同时发布的还有 **GPT-5.4 Pro**，面向需要复杂任务最高性能的用户。

GPT-5.4 将 OpenAI 近期在推理、编程和智能体（agentic）工作流方面的最佳进展整合到一个前沿模型中。它继承了 GPT-5.3-Codex 业界领先的编程能力，同时优化了跨工具、软件环境和专业任务（涉及电子表格、演示文稿和文档）的工作方式。结果是：这个模型能够准确、高效地完成复杂的实际工作，减少反复沟通，直接交付你想要的结果。

## 核心改进亮点

### ChatGPT 中的思考规划功能

在 ChatGPT 中，GPT-5.4 Thinking 现在可以在开始时提供**思考计划**，让你能够**在响应过程中调整方向**，从而更接近你需要的最终输出，无需额外的对话轮次。此外，GPT-5.4 Thinking 还改进了**深度网络研究**能力，特别是针对高度具体的查询，同时在需要更长思考时间的问题上**更好地保持上下文**。

### 原生计算机操作能力

在 Codex 和 API 中，GPT-5.4 是 OpenAI 首个具有原生、最先进**计算机操作（computer-use）能力**的通用模型，使智能体能够操作计算机并在应用程序之间执行复杂工作流。它支持高达 **1M token 的上下文**，允许智能体在长时间跨度内规划、执行和验证任务。

### Tool Search：工具使用效率革命

GPT-5.4 引入了 **tool search** 功能，这是工具使用方式的重大改进。之前，当模型被赋予工具时，所有工具定义都会预先包含在提示词中。对于拥有许多工具的系统，这可能会在每个请求中增加数千甚至数万个 token，增加成本、减慢响应速度。

有了 tool search，GPT-5.4 只接收一个轻量级的可用工具列表和工具搜索能力。当模型需要使用工具时，它可以查找该工具的定义并即时将其附加到对话中。这种方法显著减少了工具密集型工作流所需的 token 数量并保留缓存，使请求更快、更便宜。

在测试中，使用 Scale 的 MCP Atlas 基准测试的 250 个任务，tool search 配置将总 token 使用量减少了 **47%**，同时达到相同的准确率。

## 基准测试表现

| 测试项目 | GPT-5.4 | GPT-5.3-Codex | GPT-5.2 |
|---------|---------|---------------|---------|
| GDPval（胜负或平局） | 83.0% | 70.9% | 70.9% |
| SWE-Bench Pro（公开） | 57.7% | 56.8% | 55.6% |
| OSWorld-Verified | 75.0% | 74.0%* | 47.3% |
| Toolathlon | 54.6% | 51.9% | 46.3% |
| BrowseComp | 82.7% | 77.3% | 65.8% |

### GDPval：专业知识工作

在测试智能体跨 44 个职业产出规范知识工作能力的 GDPval 上，GPT-5.4 创下新纪录，在 **83.0%** 的比较中达到或超过行业专业人士水平，而 GPT-5.2 为 70.9%。

OpenAI 特别关注改进 GPT-5.4 创建和编辑电子表格、演示文稿和文档的能力。在模拟初级投资银行分析师可能执行的电子表格建模任务内部基准测试中，GPT-5.4 的平均得分为 **87.3%**，而 GPT-5.2 为 68.4%。

### 计算机操作与视觉

在 **OSWorld-Verified** 上（测量模型通过截图和键盘/鼠标操作导航桌面环境的能力），GPT-5.4 达到最先进的 **75.0%** 成功率，远超 GPT-5.2 的 47.3%，甚至超过了人类表现的 72.4%。

### 网络搜索能力

在 BrowseComp（测量 AI 智能体持久浏览网络查找难以定位信息的能力）上，GPT-5.4 比 GPT-5.2 提升了 17 个百分点，GPT-5.4 Pro 更是创下 89.3% 的新纪录。

## 事实准确性提升

GPT-5.4 是 OpenAI 迄今最注重事实准确性的模型。在一组用户标记事实错误的匿名提示词上，相比 GPT-5.2，GPT-5.4 的个人陈述虚假可能性降低 33%，完整响应包含任何错误的可能性降低 18%。

## 可用性

- **API**：GPT-5.4 现已作为 `gpt-5.4` 提供，GPT-5.4 Pro 作为 `gpt-5.4-pro` 提供
- **ChatGPT**：GPT-5.4 Thinking 今天起向 ChatGPT Plus、Team 和 Pro 用户开放，替代 GPT-5.2 Thinking
- **Codex**：包含对 1M 上下文窗口的实验性支持

## 个人解读

这次 GPT-5.4 的发布有几个值得关注的亮点：

1. **原生计算机操作能力**：这是通用模型首次原生支持 computer-use，意味着开发者可以更容易地构建能真正"操作电脑"的智能体，而不需要复杂的额外封装。

2. **Tool Search 很聪明**：这个功能解决了一个实际问题——当你的 MCP 服务器有数万个 token 的工具定义时，传统方式会严重影响性能。Tool search 让模型按需加载工具定义，token 使用减少 47% 是非常显著的提升。

3. **1M 上下文窗口**：虽然还是实验性的，但对于需要处理大量代码库或文档的智能体工作流来说，这是个巨大的加分项。

4. **定价策略**：虽然单 token 价格上涨，但由于 token 效率提升，实际使用成本可能并不会增加太多。

对于开发者而言，如果你正在构建智能体应用，GPT-5.4 的这些改进（特别是 computer-use 和 tool search）值得深入体验。
